# 🏐 Telegram Бот для Организации Опросов

Телеграм-бот для автоматизации организации волейбольных игр. Бот создает опросы по расписанию, отслеживает запись игроков, формирует списки (основной состав и запасные), ведет учет баланса игроков и управляет общей кассой группы.

## 🎯 Возможности
- **Автоматические опросы**: Создание опросов по расписанию, хранящемуся в БД.
- **Интеллектуальные списки**: Автоматическое формирование списка игроков и очереди запасных.
- **Гости**: Поддержка приглашённых игроков с отдельным приоритетом, отметкой в списке и правилами бесплатных игр.
- **Учет баланса**: Система отслеживания оплат и баланса игроков в SQLite.
- **Касса (общий баланс)**: Отслеживание фактически полученных средств и оплата залов.
- **Автоматическое списание**: Списание средств с игроков без подписки для платных залов.
- **Расчёт абонемента**: Автоматический расчёт ежемесячной подписки при закрытии голосования — стоимость зала делится поровну между подписчиками.
- **Оплата залов из кассы**: Администратор оплачивает аренду залов из накопленных средств через `/pay Оплата зала`.
- **Восстановление баланса**: Команда `/restore` для корректировки баланса без влияния на кассу (для игроков, проголосовавших, но не пришедших).
- **История транзакций**: Полное логирование всех финансовых операций с информативными описаниями.
- **Поддержка Webhook и Polling**: Работа через webhook за reverse proxy или через long polling.
- **Управление через БД**: Настройка расписания опросов и списка игроков без перезагрузки бота.
- **Статистика**: Команда `/stats` показывает сводку по играм, залам и игрокам.
- **Бэкапы SQLite**: Бот создаёт snapshot БД при старте, остановке и перед изменениями через админские команды.

## 💰 Финансовая система

Бот реализует полностью автономную финансовую систему. Единственное место, где требуется участие человека — вызов команд `/pay` и `/restore`.

### Как это работает

```
Игрок приходит без абонемента
  → Бот автоматически списывает стоимость игры (например, -150₽)
  → Игрок платит администратору
  → Админ вызывает /pay 150 → баланс обнуляется, касса +150₽

Ежемесячная подписка
  → Бот запускает голосование за абонемент
  → Подписчики голосуют за нужные залы
  → При закрытии опроса: стоимость зала / кол-во подписчиков = сумма на человека
  → Сумма списывается с баланса каждого подписчика
  → Касса НЕ меняется (деньги ещё не получены)
  → Подписчики платят администратору → /pay пополняет кассу
  → Админ оплачивает зал → /pay Оплата зала списывает из кассы

Гость голосует через пересланную голосовалку
  → Бот проверяет, состоит ли пользователь в группе
  → До 09:00 МСК следующего дня гость остаётся в «Приглашённых»
  → В 09:00 МСК бот автоматически пересобирает список
  → После 09:00 МСК гость может добрать свободные места до лимита
  → Новые голоса после 09:00 добавляются в конец, даже с абонементом
  → Первые 4 игры гостя бесплатные
  → Также бесплатно, если гость добирает состав до MIN_PLAYERS

Игрок проголосовал, но не пришёл
  → Админ вызывает /restore 150 → баланс восстанавливается
  → Касса НЕ меняется (деньги не были получены)
```

### Ключевые таблицы БД

| Таблица | Назначение |
|---------|-----------|
| `players` | Игроки: ID, имя, баланс, донат мяча, абонемент, ручная отметка гостя |
| `poll_templates` | Опросы: `id`, отображаемое `name`, расписание, стоимость игры (`cost`), стоимость аренды за игру (`cost_per_game`) |
| `poll_subscriptions` | Подписки игроков на опросы по `poll_template_id` |
| `transactions` | История всех финансовых операций с `poll_template_id` и `poll_name_snapshot` |
| `games` | История открытых/закрытых regular-опросов и месячных опросов |
| `game_participants` | История составов игр: основной/резерв/приглашённые/лист ожидания, гостевой статус и причина бесплатного участия гостя |
| `monthly_poll_votes` | Голоса месячного опроса абонемента |
| `hall_payments` | Отслеживание оплаты залов по месяцам по `poll_template_id` |
| `kv_store` | Состояние бота, баланс кассы (`fund_balance`), сохранённое состояние активных опросов |

## 🚀 Быстрый старт

### 1. Клонируйте репозиторий
```bash
git clone <repository-url>
cd volleybot
```

### 2. Настройте переменные окружения
Скопируйте пример файла `.env` и отредактируйте его:
```bash
cp .env.example .env
```
Минимально необходимые параметры:
- `TELEGRAM_TOKEN` — токен от @BotFather
- `CHAT_ID` — ID группы, где будут проходить опросы
- `ADMIN_USER_ID` — цифровой ID администратора для служебных уведомлений

### 3. Установите зависимости
```bash
# С использованием универсального скрипта:
./manage.sh setup

# Или вручную через uv:
uv venv
uv sync --locked
```

### 4. Запустите бота
```bash
source .venv/bin/activate
python -m src.bot
```

## ⚙️ Настройка

### Переменные окружения (.env)
Системные настройки (токены, порты, уровни логирования) теперь хранятся в `.env`. Подробное описание всех параметров см. в [docs/CONFIGURATION.md](docs/CONFIGURATION.md).

### Расписание опросов и игроки (SQLite)
Все данные теперь хранятся в SQLite. По умолчанию используется `data/volleybot.db`; для отдельной БД можно задать `VOLLEYBOT_DB_PATH`.
- **Игроки**: Таблица `players` (ID, никнейм, баланс, отметка о донате мячей, ручная отметка гостя).
- **Опросы**: Таблица `poll_templates` (`id`, `name`, `place`, расписание, текст сообщения, время игры, `enabled`, стоимости).
- **Подписки**: Таблица `poll_subscriptions` (кто получает уведомления об опросах, связь по `poll_template_id`).
- **История и оплаты**: Таблицы `games`, `game_participants`, `monthly_poll_votes`, `transactions` и `hall_payments` хранят историю игр, голосов, платежей и составов.
- **Бэкапы**: Snapshot-файлы создаются рядом с БД в `data/backups/`; старые бэкапы очищаются автоматически.

### Гости

Гость — игрок, который голосует не из группы, например через пересланную голосовалку в личных сообщениях. Опросы создаются неанонимными, поэтому бот видит `user_id` проголосовавшего и может проверить его членство в группе через Telegram API.

Правила состава:
- Участники группы всегда имеют приоритет над гостями.
- До 09:00 МСК следующего календарного дня после открытия опроса гости остаются в отдельной категории «Приглашённые».
- В 09:00 МСК бот автоматически обновляет открытые regular-опросы и выпускает гостей в общий порядок.
- После 09:00 МСК гости могут добрать свободные места в основной состав до `MAX_PLAYERS`.
- Все новые голоса после 09:00 МСК добавляются в конец текущего порядка, даже если игрок с абонементом.
- Если мест не осталось, гости остаются в резерве или листе ожидания.

Правила оплаты:
- Первые 4 участия гостя в `main`/`reserve` бесплатные.
- Гость также играет бесплатно, если попадает в позиции `1..MIN_PLAYERS`, то есть добирает минимальный состав.
- После первых 4 игр гость на позициях выше `MIN_PLAYERS` оплачивает обычную разовую стоимость зала.

Команда `/guest` нужна для ручной отметки гостя в базе. Это полезно, если Telegram API не смог проверить членство или если администраторам нужно заранее пометить игрока. Приглашающий не сохраняется.

Важно:
- Бот поддерживает только актуальную схему БД и больше не выполняет runtime-миграции legacy-структур.
- Если файл SQLite имеет несовместимую схему, `init_db()` завершится с `sqlite3.DatabaseError` и явным описанием расхождений.

## 🔧 Решение проблем

### Inline кнопки не работают (callback_query не приходит)

**Симптомы:**
- Команды бота работают нормально (`/start`, `/pay` и т.д.)
- Inline кнопки создаются и отображаются в Telegram
- При нажатии на кнопку ничего не происходит
- В логах нет сообщений о получении `callback_query`

**Причина:**
При установке webhook без явного указания параметра `allowed_updates`, Telegram API сохраняет список типов updates из **предыдущей** установки webhook. Если ранее webhook был установлен с ограниченным списком (например, только `["message"]`), то `callback_query` не будет отправляться.

**Решение:**
В коде `src/bot.py` при вызове `bot.set_webhook()` явно указывается список `allowed_updates`:
```python
allowed_updates = [
    "message",        # Обычные сообщения и команды
    "callback_query", # Нажатия на inline кнопки
    "poll_answer",    # Ответы на опросы
]
await bot.set_webhook(url, allowed_updates=allowed_updates)
```

**Проверка:**
Используйте команду `/webhookinfo` (только для администраторов), чтобы увидеть текущую конфигурацию webhook, включая список разрешенных типов updates.

## 🤖 Команды бота

### Для всех:
- `/help` — Показать справку по командам.
- `/schedule` — Показать расписание опросов.
- `/balance` — Проверить свой баланс.

### Для администратора:
- `/balance` — Список всех долгов + баланс кассы.
- `/subs` — Абонементы по дням.
- `/subs add HALL_ID PLAYER` — Добавить абонемент игроку на зал.
- `/stats` — Сводка за текущий месяц.
- `/stats month YYYY-MM` — Сводка за месяц.
- `/stats poll ID|Название [YYYY-MM]` — Статистика по залу.
- `/stats player ID|@username [YYYY-MM]` — Статистика по игроку.
- `/pay [сумма]` — Изменить баланс игрока (в ответ на сообщение). Касса обновляется.
- `/pay [имя] [сумма]` — Найти игрока и изменить баланс. Поиск по имени, @username или ID.
- `/pay Оплата зала` — Показать неоплаченные залы за текущий месяц и оплатить из кассы.
- `/restore [сумма]` — Восстановить баланс (в ответ на сообщение). Касса **не** меняется.
- `/restore [имя] [сумма]` — Найти игрока и восстановить баланс.
- `/player` — Список всех игроков с подробной информацией.
- `/player [имя]` — Информация об одном игроке (по имени, @username или ID).
- `/guest` — Список игроков, вручную отмеченных гостями.
- `/guest add PLAYER` — Пометить игрока гостем. `PLAYER`: ответ на сообщение, ID, @username или имя.
- `/guest remove PLAYER` — Снять ручную гостевую отметку.
- `/ball_donate` — Переключить отметку доната мяча у игрока (в ответ на сообщение).
- `/ball_donate [имя]` — Переключить донат мяча по имени, @username или ID.
- `/hall` — Список залов и команды управления расписанием.
- `/hall add` — Пошагово добавить новый зал.
- `/hall edit ID` — Пошагово изменить существующий зал.
- `/hall on ID` / `/hall off ID` — Включить или выключить зал в расписании.
- `/open_monthly` — Тестово открыть месячный опрос абонемента.
- `/close_monthly` — Закрыть месячный опрос и выполнить расчёт.
- `/start` — Включить автоматическую отправку опросов.
- `/stop` — Выключить автоматику.
- `/webhookinfo` — Проверить статус webhook и список разрешенных типов updates.

## 📁 Структура проекта
- `src/` — Исходный код бота.
  - `bot.py` — Точка входа, настройка webhook/polling.
  - `config.py` — Загрузка настроек из `.env`.
  - `db.py` — Работа с SQLite: игроки, опросы, транзакции, касса, оплата залов.
  - `handlers.py` — Обработчики команд (`/pay`, `/restore`, `/balance` и др.).
  - `poll.py` — Модели данных опросов.
  - `scheduler.py` — Планировщик опросов (APScheduler).
  - `types.py` — Типы данных (PollTemplate, HallBreakdown, SubscriptionResult).
  - `utils.py` — Вспомогательные функции.
  - `services/` — Бизнес-логика.
    - `poll_service.py` — Управление опросами, составами, гостями и расчётом абонемента.
    - `admin_service.py` — Проверка прав администратора.
    - `bot_state_service.py` — Состояние бота (вкл/выкл).
- `tests/` — Набор тестов (pytest, 335 тестов).
- `docs/` — Расширенная документация.
- `manage.sh` / `manage.ps1` — Скрипты для удобного управления проектом.
