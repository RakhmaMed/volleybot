# Руководство по конфигурации VolleyBot

Этот документ описывает все доступные параметры конфигурации бота и способы их настройки через переменные окружения.

## Способ настройки

Бот использует файл `.env` в корневой директории проекта для загрузки всех параметров. Также поддерживается установка переменных окружения напрямую.

### Начало работы

1. Скопируйте пример файла конфигурации:
   ```bash
   cp .env.example .env
   ```
2. Отредактируйте `.env`, указав ваши данные.
3. **Важно**: Никогда не добавляйте файл `.env` в систему контроля версий (git). Он уже включен в `.gitignore`.

## Параметры конфигурации (.env)

### Обязательные настройки Telegram

*   **`TELEGRAM_TOKEN`**: Токен вашего бота от [@BotFather](https://t.me/BotFather).
*   **`CHAT_ID`**: ID чата или группы, куда бот будет отправлять опросы (для групп начинается с `-100`).
*   **`ADMIN_USER_ID`**: Цифровой ID администратора Telegram для системных уведомлений и отчетов. Управление ботом доступно всем администраторам группы, которых Telegram возвращает для `CHAT_ID`.

### Настройки Webhook (опционально)

Если параметры не указаны, бот запускается в режиме **Polling** (подходит для тестов и локального запуска).

*   **`WEBHOOK_HOST`**: Публичный URL вашего сервера (например, `https://your-domain.com`).
*   **`WEBHOOK_PATH`**: Путь для вебхука (по умолчанию `/webhook`).
*   **`WEBHOOK_SECRET`**: Секретный токен для проверки подлинности запросов от Telegram.
*   **`WEBHOOK_PORT`**: Порт, который будет слушать бот (по умолчанию `8443`).
*   **`TRUST_PROXY`**: Разрешить доверие заголовкам `X-Forwarded-For` (по умолчанию `false`). Установите `true` при работе за Nginx.

При установке webhook бот явно разрешает типы updates `message`, `callback_query` и `poll_answer`, чтобы inline-кнопки и ответы на опросы продолжали приходить после переустановки webhook.

### Настройки приложения

*   **`MIN_PLAYERS`**: Минимальное количество игроков (по умолчанию `12`).
*   **`MAX_PLAYERS`**: Максимальное количество игроков в основном составе (по умолчанию `18`).
*   **`RESERVE_PLAYERS`**: Количество мест для запасных игроков (по умолчанию `3`).
*   **`POLL_OPTIONS`**: Варианты ответов в опросе через запятую (по умолчанию `Да,Нет`).
*   **`SCHEDULER_TIMEZONE`**: Часовой пояс для планировщика (по умолчанию `UTC`).
*   **`LOG_LEVEL`**: Уровень логирования (`DEBUG`, `INFO`, `WARNING`, `ERROR`).
*   **`PAYMENT_NAME`**: Имя получателя перевода, добавляется в финальные сообщения при списаниях.
*   **`PAYMENT_BANK`**: Банк или способ перевода.
*   **`PAYMENT_PHONE`**: Телефон для перевода.

### Настройки базы данных

*   **`VOLLEYBOT_DB_PATH`**: Путь к SQLite БД. Если не задан, используется `data/volleybot.db` в корне проекта.

Бэкапы создаются рядом с активной БД в папке `backups`. Например, для `data/volleybot.db` snapshot-файлы будут лежать в `data/backups/`. Бот делает бэкап при старте, остановке и перед изменениями через админские команды (`/hall`, `/subs`, `/guest`, `/start`, `/stop`, закрытие опросов).

## Конфигурация опросов (База Данных)

В отличие от системных настроек, **расписание опросов теперь хранится в базе данных SQLite** (`data/volleybot.db` по умолчанию или путь из `VOLLEYBOT_DB_PATH`).

Это позволяет изменять расписание "на лету" без перезагрузки бота.
При этом бот не выполняет runtime-миграции устаревших схем: если структура таблиц не совпадает с актуальной, `init_db()` завершится с `sqlite3.DatabaseError`.

### Параметры шаблона опроса в БД:
*   **`id`**: Стабильный числовой идентификатор шаблона.
*   **`name`**: Уникальное отображаемое название опроса.
*   **`place`**: Место проведения (текстовое поле).
*   **`message`**: Текст сообщения, который будет отправлен в чат.
*   **`open_day`**: День недели для открытия (mon, tue, wed, thu, fri, sat, sun или `*`).
*   **`open_hour_utc` / `open_minute_utc`**: Время открытия опроса в формате UTC.
*   **`game_day`**: День проведения игры (день, когда опрос автоматически закроется).
*   **`game_hour_utc` / `game_minute_utc`**: Время начала игры в UTC (опрос закрывается за 30 минут до этого времени).
*   **`cost`**: Стоимость разовой игры.
*   **`cost_per_game`**: Стоимость аренды зала за одну игру.
*   **`enabled`**: Включён ли зал в расписание.

Месячная аренда рассчитывается динамически: `cost_per_game × количество игровых дней в месяце`.

### Как управлять опросами
Администраторы управляют залами через команду `/hall`:

*   `/hall` или `/hall list` — показать список залов.
*   `/hall add` — пошагово добавить новый зал.
*   `/hall edit ID` — пошагово изменить существующий зал.
*   `/hall on ID` / `/hall off ID` — включить или выключить зал в расписании.

Для добавления абонемента игроку используйте `/subs add HALL_ID игрок`, где `HALL_ID` берётся из `/hall`.

Связанные таблицы `poll_subscriptions`, `hall_payments` и `transactions` используют `poll_template_id`, поэтому переименование `name` больше не требует ручного обновления внешних ссылок.

## Работа со временем (UTC)

Поля времени в таблице `poll_templates` хранятся в **UTC (Coordinated Universal Time)**: `open_hour_utc`, `open_minute_utc`, `game_hour_utc`, `game_minute_utc`.

Сам планировщик использует `SCHEDULER_TIMEZONE`. В `fly.toml` для production задан `Europe/Moscow`, но значения времени в БД всё равно ожидаются в UTC.

**Пример для Москвы (MSK, UTC+3):**
*   Если вы хотите открыть опрос в 18:30 по Москве, укажите:
    *   Час: `15`
    *   Минуты: `30`

## Безопасность

1. **Защита данных**: База данных `volleybot.db` содержит балансы игроков и их ID. Ограничьте доступ к этому файлу на сервере.
2. **Секреты**: Никогда не передавайте ваш `TELEGRAM_TOKEN` и содержимое `.env` третьим лицам.
3. **Webhook**: Используйте `WEBHOOK_SECRET` и закрывайте внутренний порт бота от внешнего доступа.

## Настройка за Nginx (Reverse Proxy)

Рекомендуемая схема развертывания для максимальной безопасности.

### 1. Настройка .env
```env
WEBHOOK_HOST=https://your-domain.com
WEBHOOK_PATH=/webhook
TRUST_PROXY=true
WEBHOOK_SECRET=ваш_секретный_токен_тут
```

### 2. Настройка Nginx
Создайте конфиг `/etc/nginx/sites-available/volleybot`:

```nginx
server {
    server_name your-domain.com;
    listen 443 ssl;

    location /webhook {
        proxy_pass http://127.0.0.1:8443;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # SSL сертификаты (managed by Certbot)
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
}
```

### 3. Безопасность портов
Убедитесь, что порт бота (`8443`) закрыт снаружи и доступен только локально. При использовании Docker с нашими скриптами управления (`manage.sh`), это настраивается автоматически через привязку к `127.0.0.1`.

## Docker и Fly.io

### Локальный Docker

```bash
./manage.sh deploy
./manage.sh logs
```

Скрипт монтирует:
*   `.env` в `/app/.env`
*   `data/` в `/app/data`

### Fly.io

Несекретные переменные окружения заданы в `fly.toml`. Секреты (`TELEGRAM_TOKEN`, `CHAT_ID`, `ADMIN_USER_ID`, `WEBHOOK_SECRET`, реквизиты оплаты) задавайте через Fly secrets.

Для работы с SQLite на Fly используйте:

```bash
./manage.sh db-pull [локальный-путь] [удалённый-путь]
./manage.sh db-push [локальный-путь] [удалённый-путь]
```

По умолчанию удалённый путь БД: `/app/data/volleybot.db`, локальный путь: `data/volleybot.db`. Перед перезаписью существующей БД скрипты создают резервную копию.
