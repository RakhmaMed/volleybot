# Быстрое тестирование Volleybot

## Быстрый старт

```bash
python test_bot.py
```

Что делает `test_bot.py`:
- создаёт отдельную временную БД через `VOLLEYBOT_DB_PATH`
- поднимает бота с реальными хендлерами и планировщиком
- заполняет БД тестовыми залами, игроками, подписками и кассой
- удаляет БД после `Ctrl+C`

Предустановленные данные:

| Сущность | Значение |
|---|---|
| Понедельник | 150₽ за игру, 6000₽ в месяц, подписчики: Алиса, Карина |
| Среда | бесплатно |
| Пятница | 150₽ за игру, 4500₽ в месяц, подписчики: Борис, Карина |
| Алиса | баланс `-300₽` |
| Борис | баланс `-150₽` |
| Карина | баланс `0₽` |
| Касса | `2500₽` |

## Что проверить сначала

После запуска откройте чат с ботом или тестовую группу и пройдите базовый набор:

```text
/start
/help
/schedule
/balance
/subs
/player
```

Ожидания:
- `/schedule` показывает 3 тестовых зала
- `/balance` показывает долги и кассу, если вы админ
- `/subs` показывает предзаполненные подписки
- `/player` показывает игроков и их балансы

## Финансовые сценарии

### 1. Оплата долга

```text
/balance
/pay Алиса 300
/balance
```

Ожидание:
- баланс Алисы меняется с `-300₽` на `0₽`
- касса меняется с `2500₽` на `2800₽`

### 2. Восстановление без изменения кассы

```text
/balance
/restore Борис 150
/balance
```

Ожидание:
- баланс Бориса меняется с `-150₽` на `0₽`
- касса не меняется

### 3. Оплата зала из кассы

```text
/pay Оплата зала
```

Ожидание:
- бот показывает inline-кнопки для неоплаченных платных залов
- должны быть доступны `Понедельник` и `Пятница`
- после нажатия одной кнопки касса уменьшается, а повторная оплата этого же зала за текущий месяц блокируется

Проверка:

```text
/balance
/pay Оплата зала
```

После оплаты `Понедельника` повторный список должен содержать только `Пятницу`.

## Месячный опрос

Проверка расчёта абонемента:

```text
/open_monthly
```

Дальше:
- проголосуйте за один или несколько залов
- лучше использовать несколько аккаунтов, если хотите проверить деление стоимости между подписчиками

Завершение:

```text
/close_monthly
/balance
/subs
```

Ожидание:
- месячный опрос закрывается вручную
- бот считает стоимость на человека по залам
- с проголосовавших списывается сумма
- касса при этом не меняется

Важно:
- тестируйте месячный опрос, когда нет другого открытого активного опроса
- `/open_monthly` и `/close_monthly` доступны только админу

## Что ещё стоит проверить

### Webhook/updates

```text
/webhookinfo
```

Полезно, если inline-кнопки не срабатывают.

### Работа в группе

Проверьте оба режима:
- личка с ботом
- тестовая группа, где бот админ

Для группы убедитесь, что:
- бот добавлен в чат
- у бота есть админские права
- вы запускаете админские команды с аккаунта, который считается админом

## Полный smoke-тест

Если хотите быстро прогнать почти всё вручную, используйте такой порядок:

```text
/start
/help
/schedule
/subs
/player
/balance
/pay Алиса 300
/restore Борис 150
/pay Оплата зала
/balance
/open_monthly
```

Дальше:
- проголосуйте в месячном опросе
- выполните:

```text
/close_monthly
/balance
/subs
```

## Автотесты

```bash
uv run pytest -q
```

Если нужен только набор, связанный с финансами и БД:

```bash
uv run pytest tests/test_db_polls.py tests/test_fund_and_subscriptions.py -q
```

## Примечания

- `test_bot.py` не трогает основную БД.
- Внутри БД связи теперь идут через `poll_template_id`, но в интерфейсе вы по-прежнему видите человекочитаемые названия залов.
- При остановке `test_bot.py` временная БД удаляется автоматически.
